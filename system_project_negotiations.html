お客様のニーズを掴んだ直後の商談で即座に確認すべき項目（統合版）

本資料は、システム案件（電気工事・設備工事・監視制御装置工事 等）において、 ニーズヒアリング直後〜初期商談で即座に確認すべき打合せ項目を統合し、 後続の仕様書（ドラフト）作成や見積精度向上に活用できるものです。

1. 即座に確認すべき優先項目（必須）

まずは、案件の成立可能性・成立条件を早期に見極めるため、以下の4項目を優先して確認します。

分類打合せプロセス項目確認する主な内容お客様への質問例

・「段階的な切替や仮設運転などは必要になりそうでしょうか？」

3. 後続フェーズで詳細詰めを行う項目との関係

本資料で挙げた項目は、仕様書打合せプロセス全体の入口となる位置付けです。
以下のような項目は、通常は別途技術打合せや設計レビューの場で詳細詰めを行いますが、 今回確認した内容がそれらの前提条件となります。

  • 出荷前試運転調整・工場検査：試験内容・判定基準の詳細
  • 運搬・搬入・据付工事：搬入経路制約、安全対策、仮設計画
  • 提出物件：提出図書一覧、フォーマット、提出・承認フロー
  • 保証条件の詳細：保証期間、対象範囲、保守契約との関係 など

したがって、商談の初回段階では「大きな方向性と制約条件」を確認することを目的とし、 細部は「後日の技術打合せで詰めさせてください」とお伝えしながら、 今回の6項目を確実にヒアリングすることが重要です。

4. 即座に確認すべき項目と質問例

ニーズヒアリング直後の商談で確認すべき項目と、それぞれの代表的な質問例です。

======================================================================
お客様のニーズを掴んだ直後の商談で即座に確認すべき項目
======================================================================

【優先度：高】必ず確認すべき項目（4項目）
--------------------------------------------------

■ 前提条件
  確認ポイント: 契約条件の明確化
  理由: 契約範囲と責任分担を最初に明確化し、後々のトラブルを防ぐ

  打合せプロセス:
  ・見積範囲の明確化
  ・適用規格・基準の確認
  ・除外事項の合意
  ・責任分界点の設定

  【具体的な質問例】
  ✓ 今回のプロジェクトの予算規模はどの程度をお考えですか？
  ✓ 工事範囲はどこまでを想定されていますか？（設計のみ/施工込み/試運転まで）
  ✓ 既存設備の改造や撤去工事は含まれますか？
  ✓ 適用すべき特殊な規格や基準はありますか？

■ 設備概要
  確認ポイント: 技術要求の整合性
  理由: 顧客の真のニーズと既存設備状況を把握し、適切な提案の基礎とする

  打合せプロセス:
  ・設備の用途・目的の確認
  ・要求性能・仕様の確認
  ・設置場所・環境条件の確認
  ・既存設備との取合い確認

  【具体的な質問例】
  ✓ 現在お困りの具体的な課題は何ですか？
  ✓ 既存設備の仕様書や図面はございますか？
  ✓ 必要な性能や処理能力はどの程度ですか？
  ✓ 設置場所の環境条件（温度、湿度、振動など）に特殊な要求はありますか？

■ 基本構想
  確認ポイント: 実現可能性の検証
  理由: 技術的実現可能性と概算コストを初期段階で確認

  打合せプロセス:
  ・システム構成の説明
  ・制御方式・運転方法の確認
  ・安全対策・インターロックの確認
  ・将来拡張性の検討

  【具体的な質問例】
  ✓ システムの運用イメージはお持ちですか？
  ✓ 既存の監視システムとの連携は必要ですか？
  ✓ 将来的な拡張や増設の可能性はありますか？
  ✓ 運転は自動/手動のどちらを想定されていますか？

■ 納期及び工期、保証
  確認ポイント: 進捗管理方法の合意
  理由: 希望納期と予算感を確認し、実現可能な提案を行う

  打合せプロセス:
  ・マイルストーン設定
  ・クリティカルパスの確認
  ・遅延リスクと対策協議
  ・保証期間・内容の確認

  【具体的な質問例】
  ✓ いつまでに稼働開始したいですか？
  ✓ 工事可能な時間帯や曜日の制約はありますか？
  ✓ 既存設備の停止可能期間はどの程度ですか？
  ✓ 保証期間や保守契約のご要望はありますか？


【優先度：中】時間があれば確認したい項目（2項目）
--------------------------------------------------

■ 構成機器
  確認ポイント: コスト影響の確認
  理由: 主要機器の仕様とコスト影響の概要を確認

  【具体的な質問例】
  ✓ 特定メーカーや型式の指定はありますか？
  ✓ 既存機器との互換性で注意すべき点はありますか？
  ✓ 予備品の必要性はどの程度ですか？

■ 施主殿御提供内容及び御支給品
  確認ポイント: リスク分担の明確化
  理由: 既存流用品や支給品の有無を確認し、見積範囲を明確化

  【具体的な質問例】
  ✓ 既存設備で流用したいものはありますか？
  ✓ お客様側で手配される機器や材料はありますか？
  ✓ 支給品がある場合、いつ頃入手可能ですか？

【効果的な商談の進め方】
--------------------------------------------------

1. アイスブレイク（2-3分）
   - ニーズを掴んだ話題から自然に展開

2. 設備概要の深掘り（10-15分）
   - 顧客の真の課題と要望を具体化
   - 既存設備の状況確認
   - 必要な性能・機能の明確化

3. 前提条件の確認（5-10分）
   - 予算感の把握（直接聞きにくい場合は規模感から推測）
   - 工事範囲の明確化
   - 責任分界点の初期確認

4. 基本構想の提案（10-15分）
   - 課題解決のアプローチを提示
   - 複数案の提示で選択肢を提供
   - 技術的な実現可能性を説明

5. 納期の確認（5分）
   - 希望納期と必達納期の確認
   - 工事制約条件の把握

6. 次回アクションの設定（5分）
   - 詳細技術打合せの日程調整
   - 概算見積の提出時期約束
   - 必要資料のリスト化

【重要な心構え】
• この段階では詳細仕様まで詰めない（次回の技術打合せで実施）
• 顧客の期待値をコントロール（できること/できないことを明確に）
• 不明点は持ち帰り検討とし、無理な即答は避ける
• 競合他社の動向を探りつつ、差別化ポイントを訴求
